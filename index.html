Ansible Network Configuration Automation Playbook

By Thanura Rukshan

Overview

This documentation provides a detailed guide on automating network configurations using Ansible.

Prerequisites

  • Ansible Installation: Install Ansible on the control node.
    sudo apt update && sudo apt install ansible -y  # Ubuntu/Debian
sudo yum install ansible -y  # CentOS/RHEL
  • SSH Access: Ensure SSH access is configured.
    ssh-copy-id user@target-server
  • Sudo Privileges: The Ansible user should have sudo privileges.
  • Inventory File: Define target servers in playbooks/dev.ini.
  • Python and Dependencies: Ensure Python and pip are installed.
    sudo apt install python3 python3-pip -y

Inventory File (playbooks/dev.ini)

[all]
testserver ansible_host=192.168.1.100 ansible_user=admin ansible_ssh_pass=your_password

Running the Playbook

  1. Edit dev.ini to define target servers:
    vi playbooks/dev.ini
  2. Edit site.yml to select roles:
    vi playbooks/site.yml
  3. Run the playbook:
    ansible-playbook -i playbooks/dev.ini playbooks/site.yml --ask-become-pass
  4. Run the playbook with extra verbosity:
    ansible-playbook -i playbooks/dev.ini playbooks/site.yml --ask-become-pass -vvv
